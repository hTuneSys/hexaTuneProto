# hexa-tune-proto-ffi

FFI adapter crate for the **hexaTune** protocol — C ABI functions for
Flutter/Dart and desktop integration on top of
[`hexa-tune-proto`](../hexa-tune-proto/README.md).

> `std` · cdylib + staticlib · cbindgen

## Overview

This crate exposes the hexaTune protocol as `extern "C"` functions that can be
called from any language with C FFI support — most notably Dart via `dart:ffi`.
All functions use caller-owned buffers and return integer status codes.

## Build

```bash
just build ffi        # Release build → libhexa_tune_proto_ffi.so / .dylib / .dll
just gen-header       # Regenerate C header
```

### Output artifacts

| Platform | Library file                              |
|----------|-------------------------------------------|
| Linux    | `libhexa_tune_proto_ffi.so`               |
| macOS    | `libhexa_tune_proto_ffi.dylib`            |
| Windows  | `hexa_tune_proto_ffi.dll`                 |
| iOS      | `libhexa_tune_proto_ffi.a` (staticlib)    |
| Android  | Cross-compile per ABI                     |

The generated C header is at `include/hexa_tune_proto.h`.

## API Functions

All functions follow the same convention:
- **Return `>= 0`** → success (byte count or packet count)
- **Return `< 0`** → error: `-(ProtoError code)`
- **Caller-owned buffers** — Rust never allocates

| Function                | Description                                |
|-------------------------|--------------------------------------------|
| `htp_at_encode`         | Encode AT command → byte buffer            |
| `htp_at_parse`          | Parse AT string → `HtpAtParseResult`       |
| `htp_sysex_frame`       | Wrap payload in SysEx (F0…F7)              |
| `htp_sysex_unframe`     | Extract payload from SysEx                 |
| `htp_usb_packetize`     | SysEx → 4-byte USB MIDI packets            |
| `htp_usb_depacketize`   | USB MIDI packets → SysEx bytes             |
| `htp_encode_to_packets` | Full pipeline: AT string → USB packets     |

## Dart Integration Example

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart';

final lib = DynamicLibrary.open('libhexa_tune_proto_ffi.so');

typedef HtpSysexFrameNative = Int32 Function(
  Pointer<Uint8>, Uint32, Pointer<Uint8>, Uint32,
);
typedef HtpSysexFrameDart = int Function(
  Pointer<Uint8>, int, Pointer<Uint8>, int,
);

final htpSysexFrame = lib.lookupFunction<
    HtpSysexFrameNative, HtpSysexFrameDart>('htp_sysex_frame');
```

### Error handling

```dart
final result = htpSysexFrame(payloadPtr, payloadLen, outPtr, outCap);
if (result < 0) {
  final errorCode = -result;
  throw Exception('Protocol error: code $errorCode');
}
final bytesWritten = result;
```

## C Header

The header is auto-generated by [cbindgen](https://github.com/mozilla/cbindgen)
during `cargo build`. Configuration is in `cbindgen.toml`.

Key types exported:

```c
typedef struct HtpAtParseResult { /* offsets into input buffer */ } HtpAtParseResult;

int32_t htp_at_encode(const uint8_t *name_ptr, uint32_t name_len,
                      uint8_t op, uint32_t id,
                      const uint8_t *params_ptr, uint32_t params_len,
                      uint8_t *out_ptr, uint32_t out_cap);

int32_t htp_sysex_frame(const uint8_t *payload_ptr, uint32_t payload_len,
                        uint8_t *out_ptr, uint32_t out_cap);
```

See `include/hexa_tune_proto.h` for the full generated header.
