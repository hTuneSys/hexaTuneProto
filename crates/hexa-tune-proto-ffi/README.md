# hexa-tune-proto-ffi

FFI adapter crate for the **hexaTune** protocol тАФ C ABI functions for
Flutter/Dart and desktop integration on top of
[`hexa-tune-proto`](../hexa-tune-proto/README.md).

> `std` ┬╖ cdylib + staticlib ┬╖ cbindgen

## Overview

This crate exposes the hexaTune protocol as `extern "C"` functions that can be
called from any language with C FFI support тАФ most notably Dart via `dart:ffi`.
All functions use caller-owned buffers and return integer status codes.

## Build

```bash
just build ffi          # Host platform release build
just gen-header         # Regenerate C header
just build-android      # Cross-compile for Android (arm64, armv7, x86_64)
just build-ios          # Cross-compile for iOS (device + simulator)
```

The generated C header is at `include/hexa_tune_proto.h`.

## API Functions

All functions follow the same convention:
- **Return `0`** тЖТ success (output length written to pointer)
- **Return `< 0`** тЖТ error: `-(ProtoError code)`
- **Caller-owned buffers** тАФ Rust never allocates

| Function                | Description                                |
|-------------------------|--------------------------------------------|
| `htp_at_encode`         | Encode AT command тЖТ byte buffer            |
| `htp_at_parse`          | Parse AT string тЖТ `HtpAtParseResult`       |
| `htp_sysex_frame`       | Wrap payload in SysEx (F0тАжF7)              |
| `htp_sysex_unframe`     | Extract payload from SysEx                 |
| `htp_usb_packetize`     | SysEx тЖТ 4-byte USB MIDI packets            |
| `htp_usb_depacketize`   | USB MIDI packets тЖТ SysEx bytes             |
| `htp_encode_to_packets` | Full pipeline: AT string тЖТ USB packets     |

## Dart Bindings

Pre-built Dart FFI bindings are provided in [`bindings/dart/`](bindings/dart/README.md).

```dart
import 'package:hexa_tune_proto_ffi/hexa_tune_proto_ffi.dart';

final proto = HexaTuneProto('libhexa_tune_proto_ffi.so');
final packets = proto.encodeToPackets('FREQ', id: 5, params: ['440', '1000', '1']);
final result = proto.atParse(responseBytes);
```

## C Header

The header is auto-generated by [cbindgen](https://github.com/mozilla/cbindgen)
during `cargo build`. Configuration is in `cbindgen.toml`.

See `include/hexa_tune_proto.h` for the full generated header.
